Helm simplifies the Kubernetes deployment process by abstracting out all the complexity as a developer.



helm uninstall mydb -n dev9

helm list --namespace dev9



helm install mydb bitnami/mysql --set auth.rootPassword = test1234

helm install mydb bitnami/mysql --values ./values.yaml

helm upgrade mydb bitnami/mysql --values ./values.yaml

helm upgrade mydb bitnami/mysql --reuse-values


helm uninstall mydb  --keep-history

helm install mydb bitnami/mysql --values ./values.yaml  --dry-run

helm template mydb bitnami/mysql --values ./values.yaml

kubectl get secret mydb-secret -o yaml


helm get notes mydb

helm get values mydb

helm get values aisera-aggregate-service

helm get values aisera-aggregate-service --revision 1

helm get manifest mydb --revision 1
helm get manifest aisera-event-handler --revision 21

helm history aisera-event-handler

// Rollback to particular version
helm rollback aisera-event-handler 1

//It it's there it will do upgrade if not it will do install of chart.
helm upgrade --install mydb bitnami/mysql --values ./values.yaml


//create a chart with random name
helm install bitnami/apache --generate-name

helm install bitnami/apache --generate-name --name-template "mywebserver-{{randAlpha 7 | lower}}"

//waits for complete creation of all the resources
helm install bitnami/apache --wait

//give timeout
helm install bitnami/apache --wait  --timeout 5m30s

// --atomic will automatically roll back to a previous successful release. If the installation fails.
helm install bitnami/apache --atomic



When you create a chart it will have following files.

    Chart yaml has the metadata about our chart, charts

    Chart Folder is where the dependent Charts will be copied.

    Templates Folder has all the templates that will be used to render the Kubernetes manifest later on.

    values.yaml are where all the default values for these templates are stored, and we can override those values during installation using our own values yaml or using --set option.




//create .tgz of our chart
helm package firstchart


//first is the folder of the chart we have created.
helm lint firstchart
